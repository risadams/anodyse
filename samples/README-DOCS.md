## 🤖 How It's Generated

### Workflows

The Anodyse repository demonstrates its own CI/CD integration capability through three GitHub Actions workflows:

#### 1. **docs-on-push.yml** - On Every Commit
- **Trigger**: Push to `main` branch
- **Action**: Generates fresh documentation from all sample playbooks
- **Output**: Publishes to GitHub Pages (`/samples/`)
- **Use Case**: Ensures documentation is always up-to-date with latest samples

```yaml
on:
  push:
    branches: [main]
    paths: ['samples/**', 'anodyse/**']
```

#### 2. **docs-on-pr.yml** - Validates Changes
- **Trigger**: Pull request to `main` branch
- **Action**: Validates that documentation can be generated from proposed changes
- **Output**: Comments on PR with validation status
- **Use Case**: Prevents merging changes that break documentation generation

```yaml
on:
  pull_request:
    branches: [main]
```

#### 3. **docs-schedule.yml** - Weekly Regeneration
- **Trigger**: Every Monday 2 AM UTC (configurable)
- **Action**: Regenerates all documentation with fresh timestamp
- **Output**: Creates commit if changes detected
- **Use Case**: Keeps documentation fresh even if samples don't change

```yaml
on:
  schedule:
    - cron: '0 2 * * 1'  # Monday 2 AM UTC
  workflow_dispatch:     # Manual trigger available
```

### Copy the Workflow Files

You can use these workflows directly in your own repository:

```bash
# 1. Copy workflow files to your repo
cp .github/workflows/docs-*.yml /path/to/your/repo/.github/workflows/

# 2. Update paths if your playbooks are in different locations
# Look for: samples/ → your/playbook/directory

# 3. Commit and push
git add .github/workflows/
git commit -m "feat: add Anodyse documentation generation"
git push

# 4. Workflows will run automatically on next push/PR!
```

**docs-on-pr.yml**:

```yaml
- Add additional post-generation tasks
```


## ✅ Troubleshooting

### Workflow Not Running?
- Check workflow is enabled: `.github/workflows/` directory exists
- Verify trigger conditions (branch, paths)
- Check commit message/branch matches trigger rules

### Documentation Generation Failed?
- Playbook YAML syntax error
- Missing files referenced in playbook
- Python/Anodyse installation issue
- See workflow logs for details

### Pages Not Published?
- Verify GitHub Pages is enabled in repo settings
- Check permissions in workflow file
- Wait for workflow to complete (usually <5 min)
